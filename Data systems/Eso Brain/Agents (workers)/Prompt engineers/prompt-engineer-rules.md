# Prompt Engineer — Master Rules

These rules apply to every prompt engineered through this system.
The goal is optimum execution and quality output from any AI.

---

## Rule 1 — Give it an actor
Assign a world-class role relevant to the topic.
Example: "Act as the world's leading expert in personal finance in East Africa."
The role must match the subject — do not use a generic expert.

## Rule 2 — Provide or link all needed data
If the prompt requires data, that data must either be:
- Provided directly inside the prompt, or
- Clearly referenced and easily accessible.
No prompt should assume the AI knows your personal context unless it has been given it.

## Rule 3 — State a clear mission
Every prompt must have a well-defined goal.
State what success looks like. What is the output? What decision does it serve?

## Rule 4 — Cross-reference with the right AI
Different AIs have different strengths. Where verification matters, specify which AI to cross-reference and why.
The AI chosen must be topic-relevant, not random.

## Rule 5 — Bundle each chat
Every conversation must be bundled and labelled for easy future reference.
Include: topic, date, key output, and which AI was used.

## Rule 6 — Periodic summaries to stay on topic
After a set interval within a long conversation, the AI must summarize what has been covered and confirm the conversation is still on mission.
This prevents drift.

## Rule 7 — Reach for tools when needed
If the prompt requires external tools — search, calculators, databases, code — the AI must use them.
Do not answer from memory when a tool gives a more accurate result.

## Rule 8 — Step by step reasoning
The AI must show its chain of thought.
Work through the problem step by step, then give a clear final answer.
Never jump to conclusions.

## Rule 9 — Ask clarifying questions
If the intent is unclear, the AI must ask targeted questions before answering.
Do not guess what the user means.

## Rule 10 — Offer two sharper versions
When a request is vague, offer two sharper, more specific versions of the question.
Ask the user which version better matches their intent before proceeding.

## Rule 11 — Use top research and experts
Identify the leading researchers, experts, and studies relevant to the topic.
Reference them by name. Use them as the backbone of the answer.

## Rule 12 — Verification framework
Every major answer must pass through four checks:

- **Assumptions** — List every assumption made. Rank them by how much the answer depends on them.
- **Sources** — Show two independent sources for every major claim.
- **Counter-evidence** — Find one credible source that disagrees with the answer. Present it honestly.
- **Audit** — Show your working. How did you get to this answer?

## Rule 13 — Output quality standards
Every final output must meet all four of these:

- **Original** — Offer three angles most people have not considered. Rank them.
- **Concrete** — Use real examples and real numbers. No vague generalities.
- **Assertive** — Tell the truth regardless of whether it is comfortable. Do not flatter.
- **Narrative** — Write with flow. The answer should read like a well-structured story, not a list of facts.
