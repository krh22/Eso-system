### SYSTEM ARCHITECTURE — DATA FLOW & COMPONENTS

**System Overview:**
ESO Hub is a PWA (offline-first Life OS). Frontend: GitHub Pages. Backend: PocketBase on Railway. Offline sync: localStorage queue. Hourly backup: PowerShell → GitHub.

**Core Components:**

| Component | File | Responsibility |
|-----------|------|----------------|
| UI | index.html | All panels, navigation, mode switcher |
| Config | config.js | PB_URL, collection names, AI settings |
| DB Connector | pb.js | Read/write all 6 PocketBase collections |
| Offline Sync | offline.js | localStorage queue, auto-flush on reconnect |
| AI Chat | chat.js | DeepSeek/Groq API, system prompt, responses |
| Service Worker | sw.js | Cache app shell for offline load |
| PWA Manifest | manifest.json | Phone home screen installation |
| Backup Script | backup-data.ps1 | Hourly fetch → JSON → GitHub commit |

**Data Flow — Online:** App loads → reads config.js → connects Railway → fetches data via pb.js → user writes → POST to Railway → stored in SQLite.

**Data Flow — Offline:** Service worker serves cached app → Railway unreachable → offline.js catches failed write → saves to localStorage "eso_queue" → internet returns → flushQueue() → POSTs all queued items to Railway → queue cleared.

## KEY LEARNINGS & TECHNICAL NOTES

### 🧠 KEY LEARNINGS FROM MIGRATION (Codespace → Railway)

1. **Codespace Limitations:**
   - Stops after 30 minutes of inactivity.
   - URL changes on every restart (requires config.js updates).
   - Not suitable for production or always-on services.

2. **Railway Benefits:**
   - Free tier (no credit card required for most features).
   - Permanent, fixed URL: https://pocketbase-production-e928.up.railway.app
   - 24/7 uptime (never sleeps).
   - Supports persistent storage (data survives restarts).
   - Runs PocketBase on port 8080 (not the default 8090).

3. **PocketBase v0.23.0+ Changes:**
   - Admin auth endpoint: /api/collections/_superusers/auth-with-password
   - Import endpoint: PUT /api/collections/import (not POST)
   - Import payload must be wrapped in a "collections" key.
   - Example: {"collections": [ ... ]}

4. **Import Process:**
   - Export schema from Codespace (pb_schema.json).
   - Create admin account on Railway via the installer link in logs.
   - Get admin token using the superuser auth endpoint.
   - Import schema using curl with the correct format and headers.

5. **App Update:**
   - Only change needed: Update PB_URL in config.js to the new Railway URL.
   - No changes to any other app files (pb.js, offline.js, chat.js, etc.).
   - Commit and push to GitHub → GitHub Pages automatically updates.

6. **Data Migration:**
   - Ryan confirmed he had no records in Codespace (no inbox items, no logs, no finance data).
   - Starting fresh on Railway with the same schema.
   - Data will begin accumulating from today (2026-06-26).

7. **Future Architecture:**
   - Railway can host both PocketBase and the agent backend service (Node.js/Python).
   - Same server → low latency, no additional costs.
   - Agents can be built as separate services or as endpoints within the same deployment.
   - Daily reports can be scheduled using cron jobs on Railway.

## ⚠️ KNOWN ISSUES & TROUBLESHOOTING

1. **PocketBase Admin UI not loading:**
   - Ensure you are using the correct URL: https://pocketbase-production-e928.up.railway.app/_/
   - Check Railway logs for errors.
   - Ensure the deployment is active (green status).

2. **Import errors (400 Bad Request):**
   - Use PUT method, not POST.
   - Wrap the schema array in a "collections" key.
   - Ensure the schema is valid JSON (no trailing commas).

3. **Authentication errors (401/404):**
   - Use the correct endpoint: /api/collections/_superusers/auth-with-password
   - Ensure you are using the admin email and password.
   - Get a fresh token if the previous one expired.

4. **App not loading data:**
   - Check that PB_URL in config.js is correct and points to Railway.
   - Ensure PocketBase is running and accessible.
   - Check the browser console for CORS or other errors.

5. **Offline sync issues:**
   - Ensure the service worker is registered (check browser console).
   - Check that localStorage is available and not full.
   - Test by turning off data, adding an item, then turning data back on.

6. **PWA installation issues:**
   - Use Brave or Chrome on Android.
   - Ensure the app is served over HTTPS (GitHub Pages does this).
   - Clear browser cache if the "Add to Home Screen" option does not appear.
